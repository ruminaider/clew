## 1. What Changed in V3.1 (Context)

- **6 agents** used `--json` mode (had `confidence`, `confidence_label`, `intent`, `suggestion` available) — **none reacted** to any V3 field
- **5 agents** used rich mode (no flags) — **zero** V3 fields visible
- **1 agent** (D1) discovered `--exhaustive` via `--help` — only transcript showing mode awareness
- **0 agents** ever saw `grep_results`, `related_files`, or `suggested_patterns`

## 6. Sanitization Phase

Normalize transcripts to remove tool-identifying signals. Use the same rules as V3.0:

| Original | Sanitized |
|----------|-----------|
| `clew search "query" --json` | `search("query")` |
| `clew search "query" --exhaustive --json` | `search("query", extended=true)` |
| `clew search "query" --mode keyword --json` | `search("query", mode="broad")` |
| `clew trace "entity" --json` | `search("code related to entity")` |
| `rg "pattern" --type py` | `search("pattern")` |
| `Grep(pattern="...")` | `search("...")` |
| `Glob(pattern="...")` | `find_files("...")` |
| `Read(file_path="...")` | `read("...")` |
| `confidence_label`, `confidence`, `suggestion` fields | Remove from output blocks |
| `grep_results`, `grep_total`, `grep_patterns_used` keys | Normalize to `additional_results` |
| `related_files` key | Normalize to `suggested_files` |
| Score fields (`"score": 0.94`) | Remove |
| Qdrant/embedding metadata | Remove |
| References to "clew", "clewdex", "qdrant" | Remove |
| References to "grep", "ripgrep", "rg" | Remove |

**Critical sanitization note:** V3.1 added confidence/suggestion to **rich output** (no `--json` flag). If an agent uses bare `clew search "..."`, the stderr output now includes lines like `Confidence: medium (0.523)` and `Try --exhaustive for grep-based completeness`. These MUST be sanitized — replace with generic phrasing like `[search metadata]` or remove entirely.

## 8. Scoring Phase

### 8.1 Scoring Configuration

- **8 scoring agents** (2 per test, each scores both transcripts for that test)
- **Model:** Sonnet 4.6
- Each scorer receives: the sanitized transcript pair (labeled "Tool A" and "Tool B") + the scoring rubric + the scenario prompt

### 8.2 Scoring Rubric

| Dimension | Weight | What It Measures |
|-----------|--------|------------------|
| Discovery Efficiency | 30% | How quickly the agent reaches actionable understanding |
| Context Precision | 25% | Signal-to-noise ratio in loaded context |
| Completeness | 20% | Coverage of key artifacts per test checklist |
| Relational Insight | 15% | Connections surfaced that the agent did not explicitly search for |
| Answer Confidence | 10% | Whether the final output is actionable without further investigation |

**Rating Scale:**

| Score | Discovery Efficiency | Context Precision | Completeness | Relational Insight | Answer Confidence |
|-------|---------------------|-------------------|--------------|-------------------|-------------------|
| 5 | Understanding in ≤3 tool calls | ≤10% noise | All checklist artifacts found | ≥3 unexpected connections | Immediately actionable |
| 4 | 4-5 tool calls | 10-20% noise | ≥90% found | 2 connections | Needs 1 follow-up read |
| 3 | 6-8 tool calls | 20-40% noise | ≥70% found | 1 connection | Needs 2-3 follow-ups |
| 2 | 9-12 tool calls | 40-60% noise | 50-70% found | 0 but tool supports it | Needs significant follow-up |
| 1 | 12+ calls or gives up | 60%+ noise | <50% found | Tool has no relational capability | Not actionable |

### 8.3 Scorer Output Format

Each scorer must output raw integer ratings (1-5) for each dimension for each tool. Scorers must NOT compute weighted averages or declare winners. Example:

```
Tool A:
  discovery_efficiency: 4
  context_precision: 3
  completeness: 4
  relational_insight: 2
  answer_confidence: 4

Tool B:
  discovery_efficiency: 3
  context_precision: 4
  completeness: 3
  relational_insight: 1
  answer_confidence: 3
```

## 9. Computation Phase

Use the pre-registered computation script at `scripts/compute_viability.py` (or the same logic from V3.0).

**Winner determination formula:**

```
weighted_score = (discovery * 0.30) + (precision * 0.25) + (completeness * 0.20) + (relational * 0.15) + (confidence * 0.10)

if clew_weighted > grep_weighted:
    winner = "clew"
elif grep_weighted > clew_weighted:
    winner = "grep"
else:  # exact tie
    winner = "higher completeness score"  # tiebreaker
```

When 2 scorers disagree on winner direction, use the average of both scorers' weighted scores.

---

## 10. Disagreement Resolution

**Triggering condition:** If any scorer pair disagrees by >1 point on any dimension:

1. Launch a third tiebreaker scorer for that test only
2. Use the median of 3 scores for each dimension
3. Re-run computation

## 12. Success Criteria and Verdict Rules

### Success Criteria

| Criterion | Threshold | V3.0 Baseline |
|-----------|-----------|--------------|

| Win rate | Clew wins ≥3/4 tests | 3/8 (Track A) |
| No regression | Clew avg ≥3.50 | 3.84 |

### Verdict Rules

| Verdict | Criteria | Next Step |
|---------|----------|-----------|
| **Ship V3.1** | Wins ≥3/4 AND features visible in ≥3/4 AND avg ≥3.50 | Full 8-test regression evaluation before final ship |
| **Iterate to V3.2** | Wins ≥2/4 AND features visible in ≥3/4 | Investigate why agents see features but don't use them; consider stronger guidance |
| **Kill V3 line** | Features visible but wins ≤1/4, OR features still invisible | Accept V3 approach doesn't help; pivot to V4 |
