## 4. Execution Summary

The test plan defines the full pipeline. This is a condensed overview for planning.

### Pipeline Overview

```
Phase 0: Pre-flight (5 min, manual)
  ├──→ Phase 1A: Track A exploration (8 tests × 2 agents = 16 agents)
  └──→ Phase -1: Track B scenario validation (1-2 agents)
         └──→ Phase 1B: Track B exploration (4 tests × 2 agents = 8 agents)
                  ↓ (both tracks complete)
Phase 2: Sanitization (12 agents, 1 per test)
  → Phase 3: Verification (1 agent)
    → Phase 4: Scoring (24 agents, 2 per test)
      → Phase 5: Computation (script — no agents)
        → Phase 5.5: Post-hoc feature analysis (1-2 agents)
          → Phase 6: Disagreement resolution (0-12 agents, conditional)
```

### Two Tracks

| Track | Tests | Purpose | Clew Agent Setup |
|-------|-------|---------|-----------------|
| **A: Regression** | 8 reused V2.3 scenarios (A1-C2) | Verify V3.0 default mode ≥ V2.3 quality | Default mode, no hints |
| **B: Modes** | 4 new scenarios (D1-D4) | Demonstrate V3.0 mode-specific improvement | Symmetric tool-awareness hint |

### Parallel Execution

Track A exploration and Phase -1 (Track B validation) start simultaneously. Track B exploration begins only after Phase -1 completes and checklists are sealed. This shaves validation time off the critical path.

### 5.2 Phase -1: Seal Before Explore

Track B ground-truth checklists must be sealed (written to file) before Track B exploration starts. Each scenario needs ≥5 concrete artifacts. If a scenario fails validation, replace it per the test plan's replacement protocol.

### 5.3 Track Separation During Scoring

Score Track A (A1-C2) and Track B (D1-D4) in separate scorer sessions. Don't mix tracks. Scorers for Track B tests also receive the frozen ground-truth checklist from Phase -1.

## 6. Critical Gotchas

### Target Codebase

The target is `/Users/albertgwo/Work/evvy`. Verify it exists and is indexed before starting. All exploration agents' working directory should be set to this path.

### Exploration Agent Isolation

Exploration agents must start cold with zero knowledge. They must NOT have access to:
- This handoff document
- The test plan (`docs/plans/2026-02-18-v3.0-viability-test-plan.md`)
- `CLAUDE.md` or any `.claude/` configuration
- Prior viability results
- The clew source code (they use clew as a tool, not inspect its internals)

### V3-Specific Sanitization

V3 introduces new output artifacts that need sanitization in Phase 2. The full normalization table is in the test plan (§Phase 2), but the key additions:
- `confidence_label`, `confidence`, `suggestion` → remove
- `grep_results` → normalize to `additional_results`
- `related_files` → normalize to `suggested_files`
- `--mode exhaustive` / `--exhaustive` → `search("...", extended=true)`
- `--mode keyword` → `search("...", mode="broad")`

### Verification Gate Escalation

Pre-registered 2-level escalation (not infinite re-run):
1. **First failure (>3 of 12 tests identified at >60% confidence):** Re-sanitize with stricter rules (normalize query text, remove intermediate reasoning)
2. **Second failure:** Proceed with documented caveat (consistent with V2.3 precedent)

Track B mode flags (e.g., `extended=true`) that have no grep equivalent are expected to be identifiable — note in results but don't count toward the threshold.

### Scorer Instructions

Scorers provide raw integer ratings (1-5) per dimension only. They do NOT compute weighted averages or declare winners. The exact scorer prompt is in the test plan (§Phase 4).
